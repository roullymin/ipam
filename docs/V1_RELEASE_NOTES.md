# AtlasOps v1 发布说明

发布日期：2026-04-06

## 版本定位

`v1` 是当前阶段的正式可用版本，重点完成了主路径可用性、界面统一化收口、权限感知跳转，以及核心页面的组件化拆分。

## 本版重点

### 1. 首页与导航体验

- 登录后首页升级为可执行控制台，支持快捷入口、优先事项和业务概览。
- 首页卡片支持直接跳转到目标模块，不再只是静态展示。
- 首页内容已按角色做差异化展示，降低无关信息干扰。

### 2. 全局搜索与结果落地

- 全局搜索支持更清晰的类型筛选和结果跳转。
- 搜索结果已按当前用户权限过滤，避免出现“搜得到但进不去”的情况。
- 驻场项目搜索落地逻辑已修正，支持按 `project_name` 正确进入驻场列表结果。

### 3. 列表与业务操作体验

- 抽离统一列表工具栏，统一搜索、筛选、统计和结果区样式。
- `IPAM`、`Resident`、`Change Request` 等页面已接入统一列表工具栏思路。
- 变更申请页补齐状态筛选和关键词检索，列表体验更一致。

### 4. 视觉统一化收口

- 统一卡片圆角、边框、阴影和页面密度。
- 统一页面标题区与工具栏节奏。
- 统一按钮层级、状态色和空状态表现。
- 收口了 `Dashboard`、`Security`、`Users`、`DCIM` 等页面的旧风格残留。

### 5. 组件化拆分

以下模块已完成一轮模块内组件拆分，页面文件主要负责状态与编排：

- `Dashboard`
- `Security`
- `Users`
- `IPAM`
- `Resident`
- `DCIM`

其中，`Resident` 编辑弹窗已进一步拆分为 3 个功能区组件：

- 人员信息区
- 办公与审批区
- 设备备案区

组件架构示意与职责说明见：

- `docs/V1_COMPONENT_ARCHITECTURE.md`
- `docs/assets/v1-component-architecture.svg`

## 版本与展示

- 前端展示版本统一为 `v1`
- 后端版本接口默认版本统一为 `v1`

## 验证结果

本版发版前已完成以下验证：

- 前端构建通过：`frontend/npm run build`
- 后端测试通过：`backend/python manage.py test --settings=core.settings_test`
- 后端测试数：`38`，全部通过

## 已知非阻断项

- `DatacenterChangeRequest.planned_execute_at` 仍存在 naive datetime 警告。
- 该问题当前不影响 `v1` 正常使用，但建议在 `v1.1` 中统一做时区清理。

## 建议的后续节奏

建议将当前版本定为 `v1`，后续以 `v1.1` 节奏继续推进：

- 驻场内外表单字段复用
- 更多模块接入统一工具栏和状态体系
- 时间字段时区治理
- 更细粒度的验收与使用数据回收
